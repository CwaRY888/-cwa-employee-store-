# ⚡ 3 分鐘上線 - GitHub Pages

最簡單的部署方式！跟著做就對了 🚀

---

## 📋 你需要的

- ✅ GitHub 帳號（沒有的話馬上註冊，免費的）
- ✅ 這個資料夾（代購系統）
- ✅ 瀏覽器（Chrome/Safari/Firefox）

---

## 🎯 方式一：網頁上傳（不用裝軟體）

### 步驟 1️⃣：登入 GitHub

前往: **https://github.com**

- 已有帳號 → 點「Sign in」登入
- 沒有帳號 → 點「Sign up」註冊（2 分鐘，填信箱就好）

---

### 步驟 2️⃣：建立新專案

1. 點右上角 **「+」** 
2. 選 **「New repository」**

3. 填寫:
   ```
   Repository name: cwa-employee-store
   ```
   ⚠️ 名字只能用英文、數字、連字號

4. 選擇:
   - ⭕ Public（公開，免費）
   - ⚪ Private（私有，需付費）
   
   💡 **選 Public 沒關係！**
   別人看得到網頁，但看不到員工的訂單資料。

5. 點 **「Create repository」**

---

### 步驟 3️⃣：上傳檔案

看到新頁面後：

1. 點 **「uploading an existing file」** (藍色字)

2. **拖拉整個資料夾** 進去
   
   或點 **「choose your files」** 選擇檔案
   
   ⚠️ 要上傳整個資料夾裡的所有檔案！

3. 等待上傳完成（綠色進度條）

4. 捲到最下面，點 **「Commit changes」**

---

### 步驟 4️⃣：啟用 GitHub Pages

1. 點上方的 **「Settings」**（齒輪圖示）

2. 左側選單找到 **「Pages」**（要往下捲一點）

3. 看到 **「Source」** 區塊:
   - Branch: 選 **「main」**
   - 資料夾: 選 **「/ (root)」**

4. 點 **「Save」**

5. 頁面會重新整理

6. 往上捲，看到藍色框框:
   ```
   Your site is ready to be published at 
   https://你的帳號.github.io/cwa-employee-store/
   ```

7. **等待 1-2 分鐘**（GitHub 正在建置網站）

8. 重新整理頁面，藍色框框變成綠色:
   ```
   ✅ Your site is live at 
   https://你的帳號.github.io/cwa-employee-store/
   ```

---

### 步驟 5️⃣：測試網站

1. **複製網址**:
   ```
   https://你的帳號.github.io/cwa-employee-store/
   ```

2. **在新分頁開啟**

3. 看到登入頁面 → ✅ 成功！

4. 測試:
   - 選擇一個員工名字
   - 登入
   - 瀏覽商品
   - 加入購物車
   - 送出訂單

---

### 步驟 6️⃣：分享給員工

複製這段訊息，改一下網址，傳給大家:

```
各位同事好！

🎉 CWA員工福利站已上線！

📱 網址: https://你的帳號.github.io/cwa-employee-store/

使用方式:
1. 開啟網址
2. 選擇你的名字
3. 開始購物

💡 手機小技巧:
加入主畫面就可以像 App 一樣使用！

• iPhone: Safari 點「分享」→「加入主畫面」
• Android: Chrome 點「⋮」→「加到主畫面」

有問題歡迎詢問 😊
```

## 🎯 方式二：GitHub Desktop（更簡單）

### 步驟 1️⃣：下載軟體

前往: **https://desktop.github.com/**

點「Download」下載安裝

### 步驟 2️⃣：登入

1. 開啟 GitHub Desktop
2. 點「Sign in to GitHub.com」
3. 在瀏覽器登入
4. 點「Authorize」授權

### 步驟 3️⃣：建立專案

1. 點 **「File」** → **「New Repository」**

2. 填寫:
   ```
   Name: cwa-employee-store
   Local Path: 選擇資料夾位置
   ```

3. ✅ 勾選 **「Initialize this repository with a README」**

4. 點 **「Create Repository」**

### 步驟 4️⃣：加入檔案

1. 點 **「Show in Finder」**（Mac）或 **「Show in Explorer」**（Windows）

2. 把代購系統的所有檔案複製到這個資料夾

3. 回到 GitHub Desktop

4. 看到所有檔案出現在左側

5. 左下角填寫:
   ```
   Summary: 初始上傳
   ```

6. 點 **「Commit to main」**

### 步驟 5️⃣：上傳到 GitHub

1. 點上方 **「Publish repository」**

2. ⚠️ **取消勾選** 「Keep this code private」（除非你要付費用私有）

3. 點 **「Publish Repository」**

4. 等待上傳完成

### 步驟 6️⃣：啟用 GitHub Pages

（同方式一的步驟 4）

## 🔄 如何更新？

### 修改商品/價格:

**方式一: 在 GitHub 網站上改**

1. 前往你的專案頁面
2. 點開 `🐟 CWA代購福利站.html`
3. 點鉛筆 ✏️ 編輯
4. 修改內容
5. 捲到底點「Commit changes」
6. 等 1-2 分鐘，網站就更新了

**方式二: 用 GitHub Desktop**

1. 在本機修改檔案
2. GitHub Desktop 會自動偵測
3. 填寫 Commit message
4. 點「Commit to main」
5. 點「Push origin」
6. 完成！

## ❓ 常見問題

### Q: 網址太長怎麼辦？

A: 可以用短網址服務:
- https://reurl.cc/ (台灣)
- https://bit.ly/ (國際)

把長網址縮短成: `reurl.cc/cwa-store`

### Q: 可以改網址嗎？

A: 
- 專案名稱可以改（Settings → Repository name）
- GitHub 帳號也可以改（Settings → Account）
- 改完網址就變了

### Q: 上傳失敗？

A: 常見原因:
- 檔案太大（單檔限制 100MB）→ 刪除不需要的檔案
- 檔案名稱有特殊符號 → 改成英文
- 網路不穩 → 重試

### Q: 看到 404 錯誤？

A: 
- 等 2-3 分鐘，GitHub Pages 需要時間建置
- 確認 Settings → Pages 是否啟用
- 重新整理頁面

### Q: 員工的訂單會互相看到嗎？

A: **不會！**
- 每個人的資料存在自己的瀏覽器
- 完全獨立，不會互相影響
- 這就是為什麼不用設定資料庫這麼簡單

### Q: 那團主怎麼收集訂單？

A: 三種方式:
1. 員工各自匯出訂單 Excel 給團主
2. 員工截圖購物車給團主
3. 團主登入後用「代客下單」功能

### Q: 想要即時同步怎麼辦？

A: 升級到 Firebase 雲端版
- 參考 **🔥 Firebase 設定指南.md**
- 所有訂單會集中管理
- 團主可以即時看到
