# 第一部分：快速開始

## 1. 系統簡介

### 🎯 系統定位
CWA 階梯式報價系統是專為**批發代購、團購業務**設計的訂單管理系統，支援：
- ✅ 數量階梯式報價（買越多越便宜）
- ✅ 店家／工作室雙重價格體系
- ✅ 自動從 Excel 報價單更新商品
- ✅ 完整的客戶和訂單管理

### 🎨 主要特色

#### 階梯式報價
```
範例：進口觀賞魚
┌─────────────┬─────────┬───────────┐
│ 數量區間     │ 店家價   │ 工作室價   │
├─────────────┼─────────┼───────────┤
│ 1-5 隻      │ $350    │ $320      │
│ 6-15 隻     │ $330    │ $300      │
│ 16+ 隻      │ $310    │ $280      │
└─────────────┴─────────┴───────────┘
```

#### 雙重價格體系
- **店家客戶**：標準批發價
- **工作室客戶**：優惠批發價
- 系統自動根據客戶組別顯示對應價格

#### 自動更新商品
- 從 Excel 報價單自動讀取商品
- 一鍵更新所有商品資料
- 自動備份舊版本

## 4. 階梯式報價說明

### 📊 階梯價格結構

#### 數據格式
```javascript
{
  id: 1,
  name: "紅龍魚",
  sku: "FISH-001",
  unit: "隻",
  priceTiers: [
    {
      min: 1,              // 最小數量
      max: 5,              // 最大數量
      priceStore: 3500,    // 店家價格
      priceStudio: 3200    // 工作室價格
    },
    {
      min: 6,
      max: 15,
      priceStore: 3300,
      priceStudio: 3000
    },
    {
      min: 16,
      max: 999999,         // 無上限
      priceStore: 3100,
      priceStudio: 2800
    }
  ]
}
```

### 💰 價格計算邏輯

#### 範例說明
**商品：紅龍魚**

| 數量區間 | 店家價 | 工作室價 |
|---------|--------|---------|
| 1-5 隻  | $3,500 | $3,200  |
| 6-15 隻 | $3,300 | $3,000  |
| 16+ 隻  | $3,100 | $2,800  |

#### 計算範例

**店家客戶購買 3 隻：**
```
3 隻 × $3,500 = $10,500
```

**店家客戶購買 8 隻：**
```
8 隻 × $3,300 = $26,400
（數量落在 6-15 區間，使用該階梯價格）
```

**工作室客戶購買 20 隻：**
```
20 隻 × $2,800 = $56,000
（數量超過 16，使用最高階梯的工作室價）
```

### 🎯 自動價格匹配

系統會自動：
1. ✅ 根據購買數量查找對應階梯
2. ✅ 根據客戶組別選擇價格
3. ✅ 即時計算小計和總額
4. ✅ 購物車數量變動時重新計算

## 5. 商品管理

### ➕ 新增商品

#### 固定價格商品
1. 進入「📦 商品管理」
2. 點擊「➕ 新增商品」
3. 填寫基本資料：
   ```
   商品名稱：白蝦
   分類：海鮮
   單位：斤
   品號：SEA-001
   單價（店家）：$280
   單價（工作室）：$260
   ```
4. 點擊「儲存」

#### 階梯價格商品
1. 進入「📦 商品管理」
2. 點擊「➕ 新增商品」
3. 填寫基本資料
4. 設定階梯價格：
   ```
   階梯 1：
   數量：1 ~ 10
   店家價：$280
   工作室價：$260

   階梯 2：
   數量：11 ~ 30
   店家價：$260
   工作室價：$240

   階梯 3：
   數量：31 ~ 999999
   店家價：$240
   工作室價：$220
   ```
5. 點擊「儲存」

### ✏️ 編輯商品

1. 在商品列表找到要編輯的商品
2. 點擊「✏️ 編輯」
3. 修改資料
4. 儲存變更

### 🔄 商品狀態管理

- **上架**：客戶可見並可下單
- **下架**：客戶不可見，管理員仍可看到
- 點擊「下架/上架」按鈕切換狀態

### 📊 商品庫存

#### 庫存模式
1. **無限制**：不限制銷售數量
2. **固定庫存**：設定總庫存，售完為止

#### 設定庫存
```
總庫存：100 隻
已售出：35 隻
剩餘：65 隻
```

## 7. 訂單管理

### 📋 訂單狀態

系統支援以下訂單狀態：
- **待確認**：新訂單，等待確認

## 9. 自動更新系統

### 🚀 自動更新流程

```
放入報價單 → 雙擊啟動 → 自動處理 → 系統就緒
```

### 📋 詳細步驟

#### 1. 準備報價單
將 Excel 報價單放入 `報價單資料/` 資料夾

**Excel 必需欄位：**
- 中文俗名 / 英文俗名
- 品號（SKU）
- 學名
- 尺寸
- 供應數量
- 人工/野生
- 魚店A組-最低購買量、魚店A組-批發價
- 工作室A組-最低購買量、工作室A組-批發價
- 魚店B組、魚店C組...（以此類推）

#### 2. 執行更新
**Mac：** 雙擊 `啟動系統.command`  
**Windows：** 雙擊 `啟動系統.bat`

#### 3. 自動處理
系統會自動：
1. 🔍 尋找最新報價單（依修改時間）
2. 📊 讀取並轉換商品資料
3. 🗑️ 清空舊商品資料
4. ✨ 更新 HTML 系統
5. 💾 自動備份到「備份/」
6. 🌐 開啟瀏覽器

### ⚠️ 重要提醒

**會被清空：**
- ❌ 所有商品資料
- ❌ 商品階梯價格
- ❌ 商品庫存狀態

**不會被清空：**
- ✅ 客戶資料
- ✅ 訂單記錄
- ✅ 系統設定
- ✅ LocalStorage 資料

### 🔧 自訂報價單格式

如果您的 Excel 格式不同，可修改 `auto-update-and-start.py` 中的欄位對應：

```python
# 在 convert_quotation_to_products 函數中修改
chinese_name = str(row.get('中文俗名', '')).strip()  # 改為您的欄位名
english_name = str(row.get('英文俗名', '')).strip()
sku = str(row.get('品號', '')).strip()
# ...
```

## 11. 備份與還原

### 💾 完整備份

#### 備份內容
完整備份包含：
- ✅ 所有商品（含階梯價格）
- ✅ 所有客戶資料
- ✅ 所有訂單記錄
- ✅ 系統設定

#### 備份步驟
1. 登入管理員帳號
2. 進入「⬆️ 匯入/匯出」
3. 點擊「完整備份」
4. 系統產生 JSON 檔案
5. 下載檔案（檔名格式：`CWA_完整備份_20260430_1530.json`）

#### 備份建議
- ⏰ **定期備份**：每週至少備份一次
- 📁 **多地保存**：本地 + 雲端硬碟
- 🔐 **重要操作前**：大量匯入、刪除前先備份

### 🔄 還原備份

#### 還原步驟
1. 登入管理員帳號
2. 進入「⬆️ 匯入/匯出」
3. 點擊「還原備份」
4. 選擇 JSON 備份檔案
5. 確認還原
6. 系統自動重新載入

#### ⚠️ 還原注意事項
- 還原會**完全覆蓋**現有資料
- 建議還原前先備份當前狀態
- 還原後需重新整理頁面

### 📁 自動備份

當使用「自動更新系統」時，系統會自動備份：

**備份位置：** `備份/` 資料夾

**檔案格式：**
```
🐟 CWA 階梯式報價系統（整合版）_backup_20260430_153045.html
```

**用途：**
- 保留更新前的完整系統
- 可直接開啟使用

## 14. 常見問題 FAQ

### 🎯 階梯價格相關

#### Q1: 階梯價格會自動套用嗎？
**A:** 是的。系統會根據購買數量自動查找並套用對應的階梯價格。

#### Q2: 可以設定幾個階梯？
**A:** 無限制，可依需求設定任意數量的階梯。

#### Q3: 工作室價格一定要設定嗎？
**A:** 不一定。如果不設定，會自動使用店家價格。

#### Q4: 數量跨越多個階梯怎麼計算？
**A:** 以最終數量所在的階梯計算。

**範例：**
```
階梯設定：
1-10 個 = $100
11-30 個 = $90
31+ 個 = $80

購買 25 個：
25 個都用 $90 計算 = $2,250
（不是 10×$100 + 15×$90）
```

#### Q5: 可以混用固定價格和階梯價格嗎？
**A:** 可以。商品可以自由選擇使用固定價格或階梯價格。

#### Q6: 階梯數量區間可以有空隙嗎？
**A:** 系統允許，但不建議。

**建議設定：**
```
✅ 1-10, 11-30, 31-999999（連續）
```

**不建議但可行：**
```
⚠️ 1-10, 20-30, 40-999999（有空隙）
```

### 🔧 核心函數

#### 價格計算函數
```javascript
function getProductPrice(product, quantity) {
  // 取得用戶組別
  const group = (proxyFor || currentUser)?.group || '店家';
  
  // 如果是階梯價格商品
  if (product.priceTiers && product.priceTiers.length > 0) {
    // 查找數量對應的階梯
    const tier = product.priceTiers.find(
      t => quantity >= t.min && quantity <= t.max
    );
    
    if (tier) {
      // 根據組別返回價格
      return group === '工作室' 
        ? (tier.priceStudio || tier.priceStore)
        : tier.priceStore;
    }
    
    // 找不到階梯，使用最後一個階梯
    const lastTier = product.priceTiers[product.priceTiers.length - 1];
    return group === '工作室'
      ? (lastTier.priceStudio || lastTier.priceStore)
      : lastTier.priceStore;
  }
  
  // 固定價格商品
  return group === '工作室'
    ? (product.priceTW2 || product.price)
    : product.price;
}
```

```javascript
    ? existing.qty + quantity 
    : quantity;
  
  // 取得對應階梯價格
  const price = getProductPrice(product, actualQty);
  
  if (existing) {
    // 更新數量和價格
    existing.qty = actualQty;
    existing.price = price;
  } else {
    // 新增項目
    cart.push({
      id: product.id,
      name: product.name,
      sku: product.sku,
      qty: actualQty,
      unit: product.unit,
      price: price,
      group: (proxyFor || currentUser)?.group || '店家'
    });
  }
  
  saveCart();
  renderCart();
}
```

### 🔄 自動更新流程

```
1. 執行 auto-update-and-start.py
   ↓
2. 掃描「報價單資料/」資料夾
   ↓
3. 找到最新 Excel 檔案（依修改時間）
   ↓
4. 使用 pandas 讀取 Excel
   ↓
5. 轉換為商品 JSON 格式
   ├── 解析階梯價格（A/B/C組）
   ├── 建立商品物件
   └── 驗證資料完整性
   ↓
6. 備份現有 HTML 到「備份/」
   ↓
7. 更新 HTML 中的 products 陣列
   ↓
8. 儲存更新後的 HTML
   ↓
9. 在瀏覽器中開啟
```
